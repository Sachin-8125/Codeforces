## 📂 Repository Structure

All solutions are stored at the root level, named using the standard Codeforces problem ID format:

```
Codeforces/
├── 1095A.cpp    # Problem 1095, Division A
├── 1325B.cpp    # Problem 1325, Division B
├── 1703C.cpp    # Problem 1703, Division C
├── 1873D.cpp    # Problem 1873, Division D
└── ...
```

### File Naming Convention

| Pattern | Meaning |
|---|---|
| `{contest_id}A.cpp` | Division A — typically implementation/math |
| `{contest_id}B.cpp` | Division B — greedy / brute force |
| `{contest_id}C.cpp` | Division C — data structures / logic |
| `{contest_id}D.cpp` | Division D — advanced algorithms |

## 🚀 How to Run

### Prerequisites
- GCC compiler with C++17 support (`g++`)

### Compile & Execute

```bash
# Compile
g++ -std=c++17 -O2 -o solution 1703C.cpp

# Run
./solution
```

### With Input File

```bash
g++ -std=c++17 -O2 -o solution 1703C.cpp
./solution < input.txt
```

## 🔍 Finding a Problem

Each file is named by its Codeforces problem ID. To find the problem statement, visit:

```
https://codeforces.com/problemset/problem/{contest_id}/{division}
```

**Example:** For `1873D.cpp` → https://codeforces.com/problemset/problem/1873/D
